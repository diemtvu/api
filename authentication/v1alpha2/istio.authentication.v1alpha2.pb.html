This package defines user-facing authentication policy.

Authenticator.PrincipalType

PrincipalType defines which principal (source.principal or request.auth.principal) that the authenticator will set if succeeds.

Name Description
INVALID

Sentinel value to make sure the principal type must be set.
NONE

Does not set any principal attributes. This is often used to validate extra credentials without changing the principal outputs.
SOURCE

Output principal to source.principal
REQUEST

Output principal to request.auth.principal
BOTH

Output principal to both source.principal and request.auth.principal.

MutualTls.Mode

Defines the acceptable connection TLS mode.

Name Description
INVALID

Must never be set.
STRICT

Client cert must be presented, connection is in TLS.
PERMISSIVE

Connection can be either plain text or TLS, and client cert can be omitted.
